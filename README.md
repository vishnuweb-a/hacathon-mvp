# SentinelMind

### The Security Platform That Never Forgets

SentinelMind is a memory-native Security Intelligence Platform that transforms security incidents into organizational knowledge.

Instead of treating incidents as isolated events, SentinelMind continuously learns from investigations, postmortems, and remediation activities. Every lesson learned becomes reusable memory through **Hindsight**, allowing future incidents to benefit from past experience.

---

# The Problem

Every security incident teaches valuable lessons.

Organizations discover:

* Root causes
* Effective remediation strategies
* Investigation techniques
* Operational best practices

Unfortunately, most of this knowledge is lost once an incident is resolved.

Engineers leave.

Documentation becomes outdated.

Runbooks decay.

Future incidents often require teams to rediscover solutions they already found months ago.

Traditional security platforms store incidents.

SentinelMind stores organizational memory.

---

# Solution

SentinelMind combines:

* Incident Management
* Security Copilot
* Organizational Memory
* Threat Intelligence
* Adaptive Runbooks
* Explainable Security AI

into a single continuously learning platform powered by Hindsight.

Every incident contributes to a growing organizational knowledge base.

Future investigations become faster, more accurate, and fully explainable.

# Core Features

The Security Copilot assists analysts during investigations.

Instead of relying solely on an LLM, SentinelMind first queries Hindsight for relevant historical incidents, lessons learned, and remediation strategies.

The Copilot generates:

* Root Cause Analysis
* Recommended Actions
* Resolution Guidance
* Confidence Scores

grounded in organizational memory.

---

## Continuous Learning Engine

Every resolved incident becomes new organizational knowledge.

The Learning Engine automatically extracts:

* Root Cause
* Resolution
* Lessons Learned
* Threat Classification

and stores them inside Hindsight.

The organization becomes smarter after every incident.

# How Hindsight Powers SentinelMind

Hindsight acts as the long-term memory layer of the platform.

When incidents are resolved:

1. Knowledge is extracted.
2. Structured memories are created.
3. Memories are stored in Hindsight.

When new incidents occur:

1. SentinelMind queries Hindsight.
2. Relevant memories are retrieved.
3. Recommendations are generated using historical context.

This allows SentinelMind to remember what the organization has learned over time.

Without Hindsight:

* Knowledge remains fragmented.
* AI recommendations are generic.
* Postmortems are rarely reused.

With Hindsight:

* Knowledge persists.
* Recommendations are evidence-based.
* Organizational learning compounds over time.

# Local Setup

## Clone Repository

```bash
git clone <repository-url>
cd sentinelmind
```

## Install Dependencies

```bash
npm install
```

## Configure Environment Variables

Create:

```bash
.env.local
```

Add:

```env
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=

SUPABASE_SERVICE_ROLE_KEY=

GEMINI_API_KEY=

HINDSIGHT_API_KEY=
HINDSIGHT_BASE_URL=
```
